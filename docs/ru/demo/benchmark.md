# Бенчмарк

Запустите тест производительности `simulateBuffer()` в реальном времени в вашем браузере. Измеряет пропускную способность при разных разрешениях и для всех трёх моделей симуляции.

## Методология

Бенчмарк:

1. **Прогревает** JIT 50 итерациями на маленьком буфере
2. Для каждой комбинации разрешение × модель:
   - Создаёт случайный RGBA-буфер пикселей
   - Выполняет достаточно итераций для накопления ~500мс данных
   - Выводит **медианное** время (устойчиво к паузам сборщика мусора)
3. Показывает **время на кадр** и **пропускную способность** в мегапикселях в секунду

## Что влияет на производительность

| Фактор | Влияние |
|---|---|
| **Модель** | Brettel ~1.2× медленнее Viénot/Machado (попиксельная проекция XYZ) |
| **Разрешение** | Линейное масштабирование — 4K ~4× медленнее Full HD |
| **CPU** | V8 хорошо оптимизирует горячий цикл; Apple Silicon обычно ~2× быстрее x86 |
| **Браузер** | V8 (Chrome/Edge) и SpiderMonkey (Firefox) показывают схожие результаты; JSC (Safari) варьируется |

## Типичные результаты

*Одна контрольная точка — Apple M1, Node 22, прогретый JIT, медиана по 40 итерациям. Ваши числа будут зависеть от оборудования, браузера и того, насколько прогрет JIT. Запустите бенчмарк выше, чтобы получить цифры для вашей машины.*

| Разрешение | Пиксели | Viénot | Machado | Brettel |
|---|---|---|---|---|
| VGA (640×480) | 0.3M | ~80ms | ~85ms | ~100ms |
| Full HD (1920×1080) | 2.1M | ~590ms | ~560ms | ~660ms |
| 4K (3840×2160) | 8.3M | ~2.4s | ~2.2s | ~2.6s |

Пропускная способность примерно не зависит от разрешения и составляет **~3.5 Mpix/s** для матричных моделей (Viénot, Machado) и **~3.2 Mpix/s** для Brettel.

::: warning
Один вызов `simulateBuffer()` **не** укладывается в кадр 60fps (16.7ms) ни при каком разрешении выше нескольких сотен тысяч пикселей — JavaScript просто не может обрабатывать миллионы пикселей так быстро. Для использования в реальном времени:
- Перенесите работу в [Web Worker](/ru/guide/recipes#browser-offscreencanvas-web-worker), чтобы она не блокировала основной поток
- Используйте [WebGL shader](/ru/guide/recipes#webgl-uniform-шейдера) для обработки на GPU (в 10–100× быстрее)
- Для одноразовых преобразований (например, однократная обработка загруженного изображения) указанные выше времена нормальны — пользователь воспримет ~260ms как почти мгновенные
:::

## Код

```ts
// Simple benchmark
const pixels = new Uint8ClampedArray(1920 * 1080 * 4)
// ... fill with image data ...

const start = performance.now()
simulateBuffer(pixels, 'protanopia', { model: 'machado' })
const elapsed = performance.now() - start

console.log(`${elapsed.toFixed(1)}ms`)
```

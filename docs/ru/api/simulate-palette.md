# simulatePalette()

Симулирует целую палитру цветов за один вызов. Каждый входной цвет симулируется независимо, а тип выхода сохраняется для каждого элемента (вход в виде hex-строки даёт hex-строку на выходе; RGB-объект даёт RGB-объект). Поддерживаются смешанные палитры.

Это позволяет избежать накладных расходов на выделение памяти для каждого цвета при вызове [`simulate()`](./simulate) в цикле и является рекомендуемой точкой входа для массовых проверок доступности.

## Сигнатура

```ts
function simulatePalette<T extends string | RGB>(
  colors: readonly T[],
  type: ColorblindType,
  options?: SimulateOptions,
): T[]
```

## Параметры

| Параметр | Тип | Обязателен | Описание |
|---|---|---|---|
| `colors` | `readonly (string \| RGB)[]` | Да | Массив цветов. Каждый элемент может быть hex-строкой или RGB-объектом. |
| `type` | `ColorblindType` | Да | Тип нарушения для симуляции |
| `options.model` | `ColorblindModel` | Нет | `"machado"` (по умолчанию), `"vienot"` или `"brettel"` |
| `options.severity` | `number` | Нет | 0–1, по умолчанию `1`. Значения обрезаются до допустимого диапазона. |

## Возвращаемое значение

**Новый массив** той же длины, что и входной. Входной массив никогда не изменяется. Каждый элемент сохраняет тип соответствующего входного элемента.

## Примеры

### Симуляция палитры дизайн-системы

```ts
import { simulatePalette } from 'chromanopia'

const brand = ['#e63946', '#457b9d', '#1d3557']
const simulated = simulatePalette(brand, 'protanopia')
// → ['#6c6545', '#6b7a9f', '#263758']
```

### Смешанные типы сохраняются

```ts
const mixed = simulatePalette(
  ['#ff0000', { r: 0, g: 0, b: 255 }],
  'deuteranopia',
)
// → [ '#a29000', { r: 0, g: 66, b: 133 } ]
typeof mixed[0] // 'string'
typeof mixed[1] // 'object'
```

### С моделью и степенью

```ts
simulatePalette(brand, 'protanomaly', { model: 'brettel', severity: 0.6 })
```

### Пустая палитра

```ts
simulatePalette([], 'protanopia') // → []
```

## Поведение

- Когда `type` равен `"none"`, возвращает копии входных значений (никогда не те же ссылки — см. [`simulate()`](./simulate)).
- Эквивалентно `colors.map(c => simulate(c, type, options))`, но позволяет избежать создания одноразового 4-байтового буфера пикселя для каждого цвета.
- Для пути `"none"` и RGB-входов каждый выходной элемент — это свежая копия, поэтому изменение результата никогда не затрагивает входные данные вызывающего.

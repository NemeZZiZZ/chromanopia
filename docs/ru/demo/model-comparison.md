## Ключевые различия

### Почему модели дают разные результаты?

Каждая модель использует свой математический подход к симуляции потери колбочек:

- **Viénot** проецирует на сокращённое цветовое пространство с помощью одной предвычисленной матрицы. Значения всегда неотрицательны, поэтому результат всегда находится в гамуте, но точность страдает.

- **Machado** моделирует спектральный сдвиг затронутого типа колбочек. Значения матрицы могут быть отрицательными (цвет может «заимствовать» из другого канала), что точнее, но может давать промежуточные значения за пределами гамута.

- **Brettel** вообще не использует одну матрицу — она проецирует каждый пиксель индивидуально в пространстве хроматичности CIE xyY вдоль линии смешения к точке смешения. Это наиболее точный метод, но ~1.2× медленнее.

### Когда различия важны?

Для большинства задач тестирования доступности UI различия незначительны. Они становятся заметны для:

- **Насыщенных цветов** (чистый красный, чистый зелёный) — Machado и Brettel дают заметно отличающиеся от Viénot результаты
- **Тританопии** — Brettel значительно точнее здесь благодаря попиксельной проекции
- **Частичных нарушений** (аномалии при severity < 1) — методы интерполяции различаются между моделями

## Код

```ts
import { simulate, getMatrix, isMatrixModel } from 'chromanopia'

// Compare all three models for one color
const models = ['vienot', 'machado', 'brettel'] as const

for (const model of models) {
  const result = simulate('#e63946', 'protanopia', { model })
  console.log(`${model}: ${result}`)

  if (isMatrixModel(model)) {
    const matrix = getMatrix('protanopia', { model })
    console.log('  matrix:', matrix)
  }
}
```

## 如何解读结果

每一行显示您的调色板经过特定缺陷类型转换后的效果：

- **绿色标记 (≥ 3:1)** — 颜色容易区分
- **红色标记 (< 2:1)** — 颜色可能被混淆；建议调整您的调色板

该比值是行中任意两种颜色之间的**最小成对对比度**。比值为 1:1 表示两种颜色完全相同。

## 设计建议

### 针对图表和数据可视化

- 相邻数据系列之间使用**至少 3:1 的对比度**
- 优先使用**亮度变化**（明暗对比）而非纯色相变化
- 添加**图案、标签或图标**作为辅助区分方式
- 首先针对**绿色弱（deuteranomaly）** 测试您的调色板（最常见，约 5% 的男性）

### 针对 UI 状态颜色

- 不要仅依靠红/绿来表示成功/错误 — 添加图标或文字
- 使用**蓝色 + 橙色**替代**绿色 + 红色**以获得更好的色觉缺陷对比度
- 确保所有状态颜色在**明度**上有差异，而非仅在色相上

### 安全的颜色组合

这些颜色对在大多数色觉缺陷类型下仍可区分：

| 颜色对 | 有效原因 |
|---|---|
| 蓝色 + 橙色 | 高亮度对比，不同的混淆线 |
| 蓝色 + 黄色 | S 锥体 vs L+M，仅对蓝色盲有问题 |
| 深蓝色 + 浅粉色 | 亮度差异在所有类型中都有效 |
| 黑色 + 任意亮色 | 无彩色 + 彩色始终有效 |

## 代码

```ts
import { simulate } from 'chromanopia'

const palette = ['#e63946', '#f4a261', '#2a9d8f', '#264653']
const type = 'deuteranopia'

const simulated = palette.map(color => simulate(color, type))
// → ['#988940', '#cebc61', '#858890', '#384053']
// 检查：其中是否有太相似的颜色？
```
